---
title: 'Get Missing Content'
description: 'Get a list of recent content from the provider to connect to a post with a missing release ID'
---

## Endpoint

`GET /public/v1/posts/{id}/missing`

## Description

When a post has been published but the platform did not return a usable post ID (the `releaseId` is set to `"missing"`), this endpoint fetches recent content from the provider so you can match and connect the correct one to the post.

The provider must implement the optional `missing` method. If the provider does not support it, an empty array is returned.

## Headers

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `Authorization` | string | Yes | Your organization API key |

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | The post ID |

## Response

Returns an array of content items:

```json
[
  {
    "id": "7321456789012345678",
    "url": "https://p16-sign.tiktokcdn-us.com/obj/cover-image.jpeg"
  },
  {
    "id": "7321456789012345679",
    "url": "https://p16-sign.tiktokcdn-us.com/obj/cover-image2.jpeg"
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | The platform-specific content ID |
| `url` | string | A thumbnail or preview URL for the content |

Returns an empty array `[]` if:
- The post does not have `releaseId` set to `"missing"`
- The provider does not implement the `missing` method
- No recent content was found

## Example

```bash
curl -X GET "https://your-postiz-instance.com/public/v1/posts/clx123abc/missing" \
  -H "Authorization: your-api-key"
```
